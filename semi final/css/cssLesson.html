CSS Selector

Selector Syntax - chain of one or more sequences of simple selectors separated by combinators.
Examples: :: after (example of pseudo element)
+ , > - combinator

Selector Group - comma - separated list of selectors representing the union of all elements. It is selected by each of the selectors in the list.
Examples: h1, h2, h3, :link, :visited, ul, p


Categories of Simple Selectors

  1. Descendant Combinator - combines two selectors such that elements matched by the second selector are selected if they have an ancestor element matching the first selector.
    2. Examples: white space, ie, space, tab, linefeed
  2. Child Combinator (>) - placed between two CSS selectors. It matches only those elements matched by the second selector that are the children of elements matched by the first.
    3. Syntax: selector1 > selector2
  3. Sibling Combinator (~) - separates two selectors and matches the second element only if it follows the first element (though not necessarily immediately), and both are children of the same parent element.
    4. Syntax: former_element ~ target_element { style properties }

CSS Simple Selectors

  1. Universal Selector (*) - it matches elements of any type. Each set of tags represents an element on the page.
    2. Syntax: *
  2. Type Selector - must match one or more HTML elements of the same name. Thus, a selector of nav would match all HTML nav elements, and a selector of ul would match all HTML unordered lists, or ul elements.
    3. Syntax: element
  3. ID Selectors - is declared using a hash, or pound symbol (#) preceding a string of characters. The string of characters is defined by the developer. This selector matches any HTML element that has an ID attribute with the same value as that of the selector, but minus the hash symbol.
    4. Syntax: #id_value
  4. Class Selectors - is the most useful of all CSS selectors. It is declared with a dot preceding a string of one or more characters
    5. Syntax: .class_name
  5. Attribute Selectors - targets elements based on the presence and/or value of HTML attributes, and is declared using square brackets:
    • [attr] - represents an element with an attribute name of attr.
    • [attr=value] - represents an element with an attribute name of attr whose value is exactly value.
    • [attr~=value] - represents an element with an attribute name of attr whose value is a whitespace-separated list of words, one which is exactly value.
    • [attr^=value] - represents an element with an attribute name of attr whose value is prefixed preceded by value.
  6. Pseudo - Classes - uses a colon character to identify a pseudo-state that an element might be in.
    7. Example: the state of being hovered, or the state of being activated
